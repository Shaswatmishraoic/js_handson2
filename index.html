        4."Print output: 
        var x = 21;
        var girl = function () {
            console.log(x);
            var x = 20;
        };
        girl ();"

The output of the provided code snippet would be undefined when it is executed in JavaScript. This is due to variable hoisting in JavaScript, which means that variable declarations are moved to the top of their scope during the compilation phase. However, the assignment of the value to the variable is not hoisted. 

        5."Print output: 
        var x = 21;
        girl ();
        console.log(x)
        function girl() {
            console.log(x);
            var x = 20;
        };
        "
        output: undefined
                21

the function hoisting in JavaScript brings the function declaration to the top, allowing it to be called before it is defined. However, the variable x declaration within the girl function is hoisted separately, and its assignment does not impact the outer x variable.

7.Write a function that accepts parameter n and returns factorial of n

        function factorial(n) {
            if (n === 0 || n === 1) {
              return 1;
            } else {
              return n * factorial(n - 1);
            }
          }
          
          // Example usage
          console.log(factorial(5));

        3."Guess the output

        var a = 10;
        (function (){
            console.log(a);
            var a = 20;
        })();"
        output: undefined

The variable a is declared and assigned a value of 10. Due to hoisting, the var a = 20 statement within the IIFE takes precedence over the outer a variable declaration. As a result, the console.log(a) statement prints undefined instead of 10.
